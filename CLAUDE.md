## Design Context

### Users
Mixed audience — startup PMs, enterprise EMs, freelancers, and anyone who needs a Gantt chart without heavyweight PM software (Jira, MS Project, Monday). They come to Gantarr because they want something fast and focused, not a platform to configure for weeks.

### Brand Personality
**Fast, focused, sharp.** Tool-like efficiency — get in, build the chart, get out. No fluff, no onboarding wizards, no "getting started" modals. The interface should feel like a well-made instrument: responsive, precise, and satisfying to use.

### Aesthetic Direction
**Reference:** Figma / Miro — creative canvas tools that are playful, colorful, and spatially oriented. The interface should feel like a creative workspace, not an enterprise dashboard.

**Theme:** Light mode. Clean, bright workspace feel — daytime productivity tool energy.

**Tone:** Sharp edges, confident spacing, strong color. Not cold or clinical — there's warmth in the palette — but the warmth comes from purposeful color use, not rounded-everything softness.

### Design Principles
1. **Canvas first** — The chart IS the product. Every pixel of chrome should justify itself against "could this be more chart instead?"
2. **Instant feedback** — Every interaction should feel immediate. Drag, and it moves. Click, and it responds. No loading states for local operations.
3. **Color with purpose** — Workstream and legend colors are the user's primary visual language. The UI chrome should stay neutral so user-chosen colors pop.
4. **Progressive complexity** — Simple on first glance, powerful on exploration. Basic operations are obvious; advanced features reveal through interaction.
5. **Spatial confidence** — Generous where it matters (chart area), tight where it doesn't (toolbars, panels). Asymmetric layout that feels designed, not generated.
